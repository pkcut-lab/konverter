# Dossier — roi-rechner

## § 1 — Tool-Mechanik-Research

### Formel-Varianten

**Variante 1 — Basis-ROI (2 Eingaben):**
```
ROI (%) = (Gewinn / Investition) × 100
```
Wobei Gewinn = Ertrag − Investition. Alternative Schreibweise: ROI = (Endwert − Anfangswert) / Anfangswert × 100. Nahezu alle deutschen Konkurrenten verwenden diese Variante als Primärformel [Quellen: 1, 2, 3, 4].

**Variante 2 — DuPont-Schema (3 Eingaben):**
```
ROI = Umsatzrendite × Kapitalumschlag
Umsatzrendite   = Gewinn / Nettoumsatz × 100
Kapitalumschlag = Nettoumsatz / Gesamtkapital
```
Entwickelt 1919 vom DuPont-Konzern. Liefert Zwischenkennzahlen (Umsatzrendite, Kapitalumschlag) als analytische Mehrwert-Schicht. Einziger DE-Konkurrent mit DuPont-Rechner: rechnerabisz.de [Quelle: 5].

**Variante 3 — Annualisierter ROI (3 Eingaben + Laufzeit):**
```
AROI (%) = [(Endwert / Anfangswert)^(1/n) − 1] × 100
```
n = Laufzeit in Jahren. Normiert ROI auf vergleichbare Jahreszinsrate — einzige sinnvolle Basis für Vergleiche zwischen Investitionen unterschiedlicher Laufzeit. Dominant bei englischsprachigen Konkurrenten (calculator.net, gigacalculator.com, accuratecalculators.com), in deutschen Tools nahezu absent [Quellen: 6, 7, 8].

**Variante 4 — Erweiterter ROI mit laufenden Kosten:**
```
ROI = (Gesamtertrag − Investition − laufende Kosten × Laufzeit) / Investition × 100
```
rechnerplus.de setzt diese Variante ein (4 Eingaben inkl. jährlicher Betriebskosten + Laufzeit) und liefert zusätzlich die Amortisationsdauer [Quelle: 4].

### Edge-Cases

| Input-Klasse | Verhalten | Nutzerentscheidung nötig |
|---|---|---|
| Investition = 0 | Division by zero → Fehler, kein ROI darstellbar | Nein (Validation: Pflicht) |
| Gewinn = 0 (Break-Even) | ROI = 0 %, Endwert = Anfangswert | Nein |
| Negativer ROI (Verlust) | Erlaubt, z.B. −35 % = 35 % Kapitalverlust | Nein |
| Sehr große Zahlen (>10^12) | JS float64 präzise bis 2^53 ≈ 9×10^15 | Nein (kein cap nötig) |
| Dezimaltrennzeichen DE | Komma als Eingabe muss auf Punkt normiert werden | Nein (Validation) |
| Leerzeichen, Tausenderpunkte | Input-Normalisierung vor Berechnung | Nein |
| Laufzeit 0 Jahre | Annualisierter ROI nicht berechenbar (^∞) | Ja: Laufzeit min = 0.01 Jahre oder Fehler |
| Annualisierter ROI bei n > 100 | Ergebnis nahe 0 %, kein mathematisches Problem | Nein |

### Präzisions-Erwartungen

Standard: 2 Nachkommastellen. Erweitert für Vergleiche: bis zu 4 NKS sinnvoll (z.B. annualisierter ROI 7.34 % vs. 7.38 %). DuPont-Zwischenwerte: 2–4 NKS [Quelle: 4, 5].

### Sprach-Varianten

DE-Markt verwendet sowohl "ROI" (dominierend, direktes Lehnwort) als auch "Kapitalrendite" (formaler), "Rendite" (umgangssprachlich), "Return on Investment" (ausgeschrieben). Im Tool: ROI als Primärbezeichnung, "Return on Investment" in SEO-Headline.

## § 5 — UX-Patterns Best-in-Class

### Beobachtete Patterns (aus Konkurrenz-Analyse)

- **Live-Berechnung on-typing:** papierkram.de berechnet live. Blitzrechner.de und rechnerplus.de erfordern Button-Click. Standard 2026: Debounce 150 ms, sofortiges Ergebnis.
- **Zwei-Felder-UX dominiert:** 8 von 10 Konkurrenten haben ≤3 Eingabefelder. Komplexere Inputs (netgrade.de, rechnerplus.de) adressieren Profi-Segment, aber hohe kognitive Last.
- **Break-Even-Status:** Nur papierkram.de zeigt explizit "Gewinn / Verlust / Break-Even" als semantisches Label — reduziert Interpretationsaufwand.
- **Visualisierung:** Nur calculator.net hat Kuchendiagramm (Invest vs. Gewinn). Grosser White-Space in DE.
- **Formel-Anzeige:** Kein Konkurrent zeigt Formel live neben dem Ergebnis ("ROI = 13.400 / 50.000 × 100 = 26,8 %"). Nur erklaerende Texte.
- **Copy-Button:** Nicht vorhanden bei einem einzigen der 10 Konkurrenten.
- **Mobile-Keyboard:** `inputmode="decimal"` ist Standard 2026. Kein Konkurrent dokumentiert explizit die Verwendung.

### Empfehlung

- **Uebernehmen:** Live-Berechnung (papierkram.de-Pattern), Break-Even-Status, 2-Feld-Einstieg als Default-Modus
- **Besser machen als alle:** Copy-Button auf Ergebnis-Zeile (unique in DE+EN), Formel-Aufschluesselung live neben Ergebnis, Annualisierter ROI fuer Laufzeit-Modus (adressiert §4 Pain 1)
- **Bewusst weglassen:** Slider-UX (netgrade.de-Pattern) — fuer ROI-Rechner zu komplex, Nutzer-Intent ist Schnell-Kalkulation

## § 8 — Edge-Cases + Validation-Regeln

### Allowed

- Positive Dezimalzahlen (z.B. 1234.56 oder 1234,56)
- Negative Gewinne / negative ROIs (Verlust-Szenario) — explizit erlaubt und als solche anzeigen
- Wert 0 fuer Gewinn (Break-Even)
- Laufzeit 0.01 bis 100 Jahre (Minimum fuer annualisierten ROI: >0)

### Blocked / Validation-Pflicht

| Input-Klasse | Validierungsregel | User-Message (DE) |
|---|---|---|
| Investition = 0 | Verboten (Division by zero) | "Bitte einen Investitionsbetrag groesser als 0 eingeben." |
| Nicht-numerische Eingabe | Verboten (NaN) | "Bitte eine gueltige Zahl eingeben." |
| Laufzeit ≤ 0 (bei Annualisierung) | Verboten | "Laufzeit muss mindestens 0,01 Jahre (ca. 4 Tage) betragen." |
| Laufzeit > 100 | Warnung, kein Block | "Sehr lange Laufzeit — annualisierter ROI naehert sich 0 %." |
| Negativer Anfangswert | Verboten | "Anfangsinvestition muss positiv sein." |
| Leerzeichen, Tausenderpunkte, DE-Komma | Normalisierung statt Block | — (auto-stripped) |

### Boundary-Tests

- 0 Investition → Fehler
- 1 EUR Investition, 1 EUR Gewinn → ROI = 100 %
- 10^9 EUR Investition → berechnet korrekt (JS float64 ausreichend)
- ROI = −100 % (Totalverlust) → anzeigen, kein Fehler
- Laufzeit = 0.01 Jahre, sehr kleine Annualisierung → Ergebnis kann astronomisch hoch sein → Warnung anzeigen

## § 9 — Differenzierungs-Hypothesen

### H1 — Annualisierter ROI mit Datumseingabe im DE-Markt einzigartig

**These:** Kittokit ist der einzige deutsche ROI-Rechner, der echten annualisierten ROI via Datumseingabe oder Laufzeit-Slider berechnet — mit verstaendlicher Formel-Aufschluesselung auf Deutsch.

**Beleg:** §2 Matrix zeigt: kein einziger DE-Konkurrent bietet echte AROI-Berechnung via Zinseszins-Formel. rechnerplus.de naehert nur. Alle EN-Anbieter mit AROI (calculator.net, gigacalculator.com, accuratecalculators.com) sind nicht auf Deutsch. §4 Pain 1 bestätigt Nutzer-Frustration ueber fehlende Zeitdimension.

**Wert-Beleg:** AROI loest das haüfigste Praezisions-Problem ("20 % ueber 5 Jahre vs. 8 % in 1 Jahr") — bisher in DE komplett unbesetzt.

---

### H2 — Formel-Transparenz als Vertrauensanker

**These:** Live-Formel-Aufschluesselung ("ROI = 13.400 / 50.000 × 100 = 26,8 %") neben dem Ergebnis adressiert direkt den "Black-Box"-Pain aus §4 — und differenziert von Vendor-Rechnern, die Ergebnisse ohne Herleitung ausliefern.

**Beleg:** §4 Pain 2 und Pain 4: Nutzer misstrauen ROI-Rechnern, wenn die Rechenwege unsichtbar sind. §3 Weakness 3 (blitzrechner.de): DuPont nur im Text, nie live sichtbar. Kein Konkurrent zeigt die Formel live neben dem Eingabe-Ergebnis.

**Wert-Beleg:** Senkung der kognitivenUnsicherheit → hoehere Nutzer-Verweildauer, SEO-Signal.

---

### H3 — Drei Modi in einem Tool (Einfach / Erweitert / DuPont)

**These:** Ein schrittweise aufklappendes Interface — Basis (2 Felder), Erweitert (+ Laufzeit + Betriebskosten → annualisierter ROI + Amortisation), DuPont-Modus (3 Felder: Gewinn+Umsatz+Kapital) — deckt mehr Intent-Segmente ab als jeder Einzelkonkurrent im DE-Markt.

**Beleg:** §2 Matrix: rechnerplus.de hat Erweitert, rechnerabisz.de hat DuPont — aber keiner bietet beides. Intent-Mix aus §6: 70 % informational (FAQs, Formel) + 30 % transactional (schnelle Berechnung) → beide Gruppen haben unterschiedliche Einstiegstiefen.

**Wert-Beleg:** Breitere Keyword-Abdeckung (roi formel + roi dupont + kapitalrendite berechnen + amortisationszeit) in einem einzigen Tool.
