# Category-Root-Dossier: finance

## 1. Kategorie-Abgrenzung

Finance-Tools auf kittokit sind reine **Gross-Berechnungstools ohne Serverrundreise** — sie rechnen im Browser, speichern nichts, benötigen keine API-Keys oder Datenbankzugriffe. Sie grenzen sich ab von:

- Brutto-Netto-Rechnern (erfordern Steuerklasse, Sozialabgaben, Jahresdetails → separater Scope)
- Echtzeitkurs-Rechnern (Forex, Crypto → erfordern API, keine Pure-Client-Lösung)
- Buchführungs- / DATEV-Tools (serverabhängig)

Erlaubte Finance-Tool-Typen: **Calculator** (Umrechnungen, Gehaltsanalysen), **Formatter** (Zahlenformate), **Analyzer** (Vergleiche, Benchmarks).

### Formeln (Autoritativ für DE-Markt)

**Schnell-Methode (einfach, 5-10% ungenauer):**
```
Jahresgehalt = Stundenlohn × Wochenstunden × 52
Stundenlohn  = Jahresgehalt / (Wochenstunden × 52)
```

**Exakt-Methode (Rentenversicherungsformel):**
```
Monatsstunden = Wochenstunden × 4 × 13 / 12
Stundenlohn   = 3 × Monatsgehalt / 13 / Wochenstunden
Jahresgehalt  = Stundenlohn × Wochenstunden × 13 / 3 × 12
```

**Mit Urlaub/Feiertagen (präziseste):**
```
Arbeitstage/Jahr = 260 − Urlaubstage − Feiertage
Jahresgehalt     = Stundenlohn × (Arbeitstage/Jahr × Stunden/Tag)
```

## 3. User-Pain-Points der Kategorie

1. **Brutto-Netto-Gap:** Nutzer wollen Nettoergebnis, Tools zeigen Brutto → Frustration und Missverständnisse beim Vergleich
2. **Formel-Verwirrung:** ÷4,33 vs. ×13/12 vs. ×52 — Ergebnisdifferenzen von 5–10% ohne Erklärung
3. **Fehlende Bundesland-Feiertage:** Durchschnittswerte statt statespecifischer Feiertage führen zu Ungenauigkeiten
4. **Keine Freelance-Sicht:** Selbstständige benötigen höheren Stundenlohn (Krankenversicherung, Altersvorsorge selbst zahlen) — kein Rechner deckt das vollständig ab

## 4. 2026-Trends (Kategorie-weit)

- **Privacy-as-Feature:** Wachsende DSGVO-Sensibilität → "Alle Berechnungen lokal im Browser" als explizit kommunizierter USP
- **Mindestlohn-Aktualität:** Jährliche Anpassungen (2026/2027 bereits beschlossen) → Tools müssen Updates automatisieren oder klar versionieren
- **4-Tage-Woche-Adoption:** Wachsendes Nutzer-Segment in DE, das 32h statt 40h als Standard hat
- **Mobile-First-Suchen:** Gehaltsrechentools werden zunehmend mobil abgerufen (on the go, während Jobverhandlung)
- **AEO/Voice-Search:** FAQ-Inhalte ("Was ist mein Stundenlohn bei X Euro Jahresgehalt?") gewinnen an SEO-Gewicht
- **Freelancer-Segment:** Wachsende Zahl Selbstständiger braucht differenzierte Rate-Kalkulatoren
