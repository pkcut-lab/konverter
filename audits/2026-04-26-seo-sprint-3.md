## T3.1 — hreflang-Audit + Fix

### Bug Fixed
`[slug].astro` was passing `pathWithoutLang="/{current-lang-slug}"` to BaseLayout, causing
cross-language hreflang to use the wrong slug:
- DE page `/de/meter-zu-fuss` → EN hreflang pointed to `/en/meter-zu-fuss` ✗
- After fix → EN hreflang points to `/en/meter-to-feet` ✓

### Fix
`pathByLang` record now holds per-language paths: current lang uses routed slug directly,
other languages looked up from slug-map via `getSlug(toolId, l)`.

### Coverage

- Bidirectional symmetry: `dePageEnHref == enPageSelfHref` for all 72 tools
- x-default always matches DE (default language) URL
- Regression test confirms old single-string bug behaviour

## T3.2 — INP-Tuning

### Changes
- `HeaderSearch client:idle` → `client:visible` in `Header.astro`
- `ThemeToggle client:idle` → `client:visible` in `Header.astro`

### Why
Both components are in-viewport on every page load. `client:visible` hydrates via
IntersectionObserver which fires immediately for visible elements — vs. `client:idle`
which waits for `requestIdleCallback`. On slow main threads, this reduces the window
where a user click on HeaderSearch/ThemeToggle goes unhandled.

### Pagefind Verification
`pagefind.test.ts` regex updated to allow `client:(load|idle|visible)` — no regression.

### INP Target
INP <200ms p75 on tool pages. Lighthouse run deferred (requires built dist + live server).
Pre-conditions improved by reducing hydration latency for above-fold interactive elements.

## T3.3 — Internal-Linking-Strategist

### New File
`src/lib/related-tools.ts` — pure `computeRelatedTools(toolId, category, all, maxCount)`:
- Scores by category match (weight 3) — same category group → related
- Symmetry guarantee: A→B ↔ B→A (same-category is transitive)
- Zero deps, no async — fully unit-testable

### Article Block
`[lang]/[slug].astro` renders a `tool-article__related` aside inside `<article>` after
prose content. Shows up to 5 computed-related tools as bordered chips. Filtered to
exclude tools already in the explicit `relatedSlugs` list to avoid duplication with the
Related-Bar at the top.

### Symmetry Tests
- 6 assertions: self-exclusion, maxCount, empty-on-no-category, cross-category isolation
- Symmetry property: for every tool A and every B in `computeRelatedTools(A)`, A appears
  in `computeRelatedTools(B)`
