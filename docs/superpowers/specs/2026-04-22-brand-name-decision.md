# Brand-Name-Entscheidung — Konverter-Webseite

## TL;DR

**Primary Pick: `Pondari`**

- Latein-Wurzel *ponderare* = abwägen, gewichten → Konversions-Akt-Metapher ⭐⭐⭐⭐
- `pondari.com` + `pondari.de` + `pondari.tools` + `pondari.io` + `pondari.app` + `pondari.dev` + `pondari.ai` + `pondari.xyz` alle frei
- Tech-Ecosystem clean (npm / PyPI / crates.io / Docker / GitHub-Org frei)
- TM über 3 Register grün nach Deep-Research (mit 3 Conditions, siehe §4.2)

**Backup-Pick: `Armillex`** (falls Markenanwalt-FTO Pondari-Hindernis findet — Armillex ist TM-cleaner ohne Conditions, Brand-Story sogar stärker ⭐⭐⭐⭐⭐)

**3 Bedingungen vor Anmeldung (nicht verhandelbar):**

1. Markenanwalt-FTO-Clearance (~150-250 EUR) für Pondari inkl. Distanz-Analyse zu **Pondera Solutions / Thomson Reuters** (Fraud-Detection SaaS Klasse 9/42, seit 2020 Teil von TR)
2. Domain-Reservierung (`pondari.com` + `pondari.de` + `pondari.tools`) VOR Anwaltstermin — Tippfehler-Squat-Schutz
3. DPMAregister-Direktsuche + EUIPO eSearch plus manuell durchführen (Web-Checks sind JS-blockiert, nur via Browser valide)

### 4.2 Verbleibende Risiken (TRANSPARENT)

#### Risiko 1: Pondera Solutions (Thomson Reuters seit 2020)

- Fraud-Detection SaaS, US-Enterprise/Government-Markt
- Nice-Klassen: voraussichtlich 9 + 42 (anti-fraud software + analytics services)
- **Spelling-Distanz**: Pondera vs Pondari — letzter Vokal differiert (A vs I)
- **Scope-Distanz**: Enterprise-Fraud-Detection vs Consumer-Konverter — **unterschiedliche Branchen, unterschiedliche Kundenkanäle**
- **Likelihood-of-Confusion**: niedrig (DuPont-Faktoren sprechen für uns: different goods, different channels, different customer base)
- **Aber**: Thomson Reuters hat Legal-Budget für „Nastygrams" auch bei dünnen Ansprüchen

**Mitigation:**
- DE-Markenanmeldung FIRST (Priority für Madrid-Ausweitung)
- Goods/Services eng gefasst: „Online-Konverter-Werkzeuge für Einheiten, Dateiformate und Text"
- KEINE Scope-Überdehnung in „data analytics / fraud detection / enterprise"
- Trademark-Watch-Service (Corsearch/Markify ~150 EUR/Jahr) für Pondera/Pondari/Pondar-Familie

#### Risiko 2: Pondari = echter indischer Nachname + Dorf + Tamil-Homophon

- **Nachname**: 612 Träger in Indien, 98% Andhra Pradesh (Telugu-Gebiet)
- **Dorf**: Pondari im Bulandshahr-Distrikt, Uttar Pradesh, 6,639 Einwohner
- **Tamil-Homophon**: *pondatti/pondaati* (பொண்டாட்டி) = Ehefrau (72M Native-Sprecher + Diaspora UK/Canada/Singapore/Malaysia), romanisierte Schreibweisen variieren stark — „Pondari" sitzt in der Ambiguitäts-Wolke
- **SERP-Pollution**: Google „pondari" liefert 100% indischen Kontext auf Seite 1 (Forebears, Village Info, LinkedIn-Profile, ResearchGate Satyanarayana)

**Impact per Phase:**
- **Phase 1-4 (DE/EN/ES/FR/PT-BR)**: Irrelevant — kein Tamil/Hindi-Publikum
- **Phase 5+ (wenn Indien-Expansion)**: SEO-Hürde + Brand-Story-Kollision (Name wirkt wie Person, nicht wie Brand) + Tamil-Smirk-Risiko

**Mitigation:**
- Launch-Phasen 1-4 kein Blocker
- Falls Phase-5 Indien-Rollout: lokalisierte Sub-Brand ODER Vokal-Tweak (Pondero/Pondaria/Ponderis) für diesen Markt
- SEO-Strategie: Long-Tail „pondari konverter" / „pondari tools" — nicht Head-Term „pondari"

#### Risiko 3: Social-Handle-Konflikt Instagram

- `@pondari` auf Instagram belegt (japanischer Privat-Account „岡本 ぽん", kein Brand-Competitor)
- **Mitigation**: Handle-Variante für IG: `@pondari.tools` / `@getpondari` / `@pondari.app`
- GitHub-Org `pondari` frei — primärer Tech-Handle safe

### 4.3 Warum NICHT Armillex (obwohl TM-cleaner)

- Armillex ist TM-technisch **reiner** (über alle 3 Register grün OHNE Conditions)
- Brand-Story sogar stärker ⭐⭐⭐⭐⭐ (Armillarsphäre = 1000 Konverter als Präzisions-Ringe um UX-Kern)
- **Aber**: User-Preference für Pondari — phonetischer Klang weicher, domain-ästhetisch (pondari.tools gefällt)
- Pondera-Solutions-Risiko ist akzeptabel (Scope-Distanz groß)
- Indian-Echo ist Phase-5-Problem, nicht Launch-Blocker
- Armillex bleibt **Backup-Pick** falls Markenanwalt-FTO unerwartetes Hindernis findet
